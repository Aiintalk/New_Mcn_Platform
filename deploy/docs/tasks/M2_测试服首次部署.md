# 运维任务单 · 测试服首次部署

> 目标：将 mcn-platform（M2 Sprint 1 版本）完整部署到测试服务器
>
> 服务器环境（已确认）：
> - Ubuntu，2核 3.4GB 内存，79GB 磁盘
> - Python 3.10.12 / Node.js 22 / PM2 v7 / Nginx 1.18 / PostgreSQL 15 / Git 2.34
> - 所有依赖已预装，无需额外安装

---

## 部署前检查清单

在开始之前，确认以下信息已准备好：

- [ ] GitHub 仓库地址（`git clone` 用）
- [ ] 数据库密码（强密码，不使用默认 admin123）
- [ ] JWT_SECRET（随机字符串，至少 32 位，可用 `openssl rand -hex 32` 生成）
- [ ] ENCRYPTION_KEY（32 位随机字符串）
- [ ] 服务器公网 IP 或域名

---

## 步骤 1 — 拉取代码

```bash
# 在服务器上执行
cd /opt
sudo git clone https://github.com/<your-org>/mcn-platform.git
sudo chown -R $USER:$USER /opt/mcn-platform
cd /opt/mcn-platform
```

---

## 步骤 2 — 配置后端环境变量

```bash
cd /opt/mcn-platform/backend
cp .env.example .env
nano .env   # 或 vim .env
```

**必须修改的字段：**

```env
# 数据库（替换为真实密码）
DATABASE_URL=postgresql+asyncpg://postgres:<强密码>@localhost:5432/mcn_db

# 安全密钥（必须修改，不能用默认值）
JWT_SECRET=<openssl rand -hex 32 生成的随机字符串>
ENCRYPTION_KEY=<另一个 32 位随机字符串>

# 管理员初始账号（首次登录后立即改密码）
INITIAL_ADMIN_USERNAME=admin
INITIAL_ADMIN_PASSWORD=<自定义强密码>
```

**AI / TikHub Key 暂时可留空**（部署后在管理后台配置 Key Pool）：
```env
LLM_API_KEY=placeholder
TIKHUB_API_KEY=placeholder
```

---

## 步骤 3 — 创建数据库

```bash
# 创建数据库用户和库
sudo -u postgres psql << 'EOF'
CREATE USER mcn_user WITH PASSWORD '<强密码>';
CREATE DATABASE mcn_db OWNER mcn_user;
GRANT ALL PRIVILEGES ON DATABASE mcn_db TO mcn_user;
EOF
```

> ⚠️ 密码要与步骤 2 中 DATABASE_URL 里的密码一致

---

## 步骤 4 — 执行数据库迁移（001 → 007）

```bash
cd /opt/mcn-platform/backend

# 按顺序执行所有迁移脚本
for f in migrations/001_init.sql \
          migrations/002_kols_add_owner.sql \
          migrations/003_ai_tables.sql \
          migrations/004_credentials_test_fields.sql \
          migrations/005_ai_models_test_fields.sql \
          migrations/006_kol_intake.sql \
          migrations/007_kol_intake_operator_sessions.sql; do
  echo "执行 $f ..."
  PGPASSWORD=<强密码> psql -h localhost -U mcn_user -d mcn_db -f $f
  echo "✅ $f 完成"
done
```

验证表数量：
```bash
PGPASSWORD=<强密码> psql -h localhost -U mcn_user -d mcn_db -c "\dt"
# 预期：应看到约 20 张表
```

## 步骤 5 — 安装后端 Python 依赖

```bash
cd /opt/mcn-platform/backend

# 创建虚拟环境（用系统 Python 3.10）
python3 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 验证
python -c "import fastapi; print('FastAPI OK')"
```

---

## 步骤 6 — 构建前端

```bash
cd /opt/mcn-platform/frontend

# 配置前端环境变量（指向服务器 IP 或域名）
cp .env.example .env
```

编辑 `.env`：
```env
VITE_API_BASE_URL=http://<服务器公网IP>
# 如果有域名则填域名：VITE_API_BASE_URL=http://your-domain.com
```

```bash
# 安装依赖并构建
npm install
npm run build

# 将构建产物复制到 Nginx 静态目录
sudo mkdir -p /var/www/mcn
sudo cp -r dist/* /var/www/mcn/
sudo chown -R www-data:www-data /var/www/mcn
```

---

## 步骤 7 — 配置 Nginx

```bash
# 复制 Nginx 配置
sudo cp /opt/mcn-platform/deploy/nginx/mcn-m1.conf /etc/nginx/sites-available/mcn

# 启用站点
sudo ln -s /etc/nginx/sites-available/mcn /etc/nginx/sites-enabled/mcn

# 禁用默认站点（避免冲突）
sudo rm -f /etc/nginx/sites-enabled/default

# 验证配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

---

## 步骤 8 — 用 PM2 启动后端

```bash
cd /opt/mcn-platform/backend
source .venv/bin/activate

# 用 PM2 启动后端（进程守护 + 自动重启）
pm2 start "python -m uvicorn app.main:app --host 127.0.0.1 --port 8000" \
  --name mcn-api \
  --interpreter none

# 查看启动状态
pm2 status

# 查看启动日志（确认无报错）
pm2 logs mcn-api --lines 50

# 设置 PM2 开机自启
pm2 save
pm2 startup
# 执行上面命令输出的那行 sudo ... 命令
```

## 步骤 9 — 健康检查

```bash
# 检查后端是否正常响应
curl http://localhost:8000/api/health
# 预期：{"success":true,"code":"OK","data":{"status":"ok","db":"ok"}}

# 检查 Nginx 是否正确转发
curl http://<服务器公网IP>/api/health
# 预期：同上

# 检查前端页面
curl -I http://<服务器公网IP>/
# 预期：HTTP 200
```

## 步骤 10 — 初始化后台配置

部署完成后，用浏览器打开 `http://<服务器IP>`，用管理员账号登录，完成以下配置：

1. **修改管理员密码**（首次登录强制修改）
2. **服务配置 → AI 模型管理** → 添加 AI Key（yunwu 或其他服务商）
3. **服务配置 → 问卷配置 → AI 配置** → 填写两段 Prompt：
   - `conversation_bridge`：对话过渡语 Prompt
   - `report_generation`：报告生成 Prompt
4. **服务配置 → TikHub** → 添加 TikHub API Key

## PostgreSQL 内存优化（3.4GB 服务器）

```bash
sudo nano /etc/postgresql/15/main/postgresql.conf
```

修改以下参数：
```conf
shared_buffers = 256MB
effective_cache_size = 1GB
work_mem = 8MB
maintenance_work_mem = 64MB
max_connections = 40
```

```bash
sudo systemctl restart postgresql
```
