### 3.2 鉴权

| 依赖函数 | 定义位置 | 用途 |
|----------|----------|------|
| `Depends(get_current_user)` | `middlewares/auth.py` | 仅需登录 + 未禁用 |
| `Depends(require_admin)` | `middlewares/auth.py` | admin 角色 + 已改密 |
| `Depends(require_operator)` | **各 router 文件内部** | operator/admin 角色 + 已改密 |
| `Depends(require_password_changed)` | `middlewares/auth.py` | 任何已改密用户 |
| `Depends(get_current_user_optional)` | `middlewares/auth.py` | 可选登录（有 token 解析，无 token 返回 None） |

注意：`require_operator` 目前定义在使用它的 router 文件内（如 `operator_homepage.py:29`），不在公共 `auth.py` 中。

**公开接口**（login）不加任何 `Depends`。

### 3.3 数据库会话

实际存在两种模式：

**模式 A — `AsyncSessionLocal()` 手动管理（多数 admin router）**

```python
async with AsyncSessionLocal() as session:
    result = await session.execute(select(User))
    await session.commit()
```

特点：session 块结束自动关闭，commit/rollback 手动控制。

**模式 B — `Depends(get_db)` 注入（部分 router）**

```python
async def my_endpoint(db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(User))
    await db.commit()
```

注意：`get_db` 是一个 generator，只 yield session，**不会自动 commit/rollback**，行为与模式 A 一致。

**推荐**：新代码用模式 A（与现有代码风格一致），测试场景可用模式 B。

### 3.4 响应

```python
# 成功
return success_response(data={"key": "value"})
return success_response(data=None, message="操作成功")

# 错误
return error_response(ErrorCode.USER_NOT_FOUND, "用户不存在")

# 鉴权层异常（直接 raise，不走 success/error_response）
raise HTTPException(
    status_code=401,
    detail={"code": ErrorCode.AUTH_TOKEN_EXPIRED, "message": "Token 已过期或无效"}
)
```

鉴权层使用 `_http_err()` 辅助函数（定义在 `middlewares/auth.py`）。

## 4. Service 模式

### 4.1 标准数据 Service（推荐）

```python
async def service_function(db: AsyncSession, ...) -> domain_object | dict:
    """接收 db 参数，不自创会话。"""
    # 查询
    result = await db.execute(select(Xxx).where(...))
    obj = result.scalar_one_or_none()

    if not obj:
        raise RuntimeError("Xxx not found")

    # 更新
    await db.execute(update(Xxx).where(Xxx.id == obj.id).values(...))
    await db.commit()  # Service 层自行 commit
    return obj
```

- **接收 `db: AsyncSession` 参数**（不自创会话）
- **返回领域对象或 dict**（不返回 HTTP 响应）
- **失败抛 `RuntimeError`**（由 router 的 `try/except` 转为 `error_response`）
- **自行 `await db.commit()`**（当前代码中 Service 层负责 commit）

### 4.2 外部服务 Service

对于外部 API 调用（如 TikHub），当前模式是返回状态 dict 而非抛异常：

```python
async def fetch_tikhub_for_kol(kol: Kol, db: AsyncSession) -> dict:
    try:
        # ... 调用外部 API ...
        await db.commit()
        return {"status": "ok", "duration_ms": ..., ...}
    except Exception as e:
        await db.commit()  # 提交错误日志
        return {"status": "error", "error": str(e), ...}
```

**推荐**：新代码的数据 Service 用 `raise RuntimeError`，外部服务调用可保留返回 dict 模式（因为需要记录错误到日志表）。

#### TikHub 调用必须经 `tikhub_adapter.*` 函数

`adapters/tikhub.py` 的 6 个公开 async 函数（`resolve_sec_user_id` / `get_user_profile` / `get_user_fans_info` / `get_live_room_products` / `fetch_user_videos` / `fetch_video_by_share_url`）已用 try/finally 模式自动写 `TikHubCallLog`（含 credential_id / user_id / endpoint / status / latency_ms / error_message）。

- ✅ 业务代码只调 `tikhub_adapter.*` 函数，凭证池选择 + 失败上报 + 日志写入全自动
- ❌ 禁止裸 `httpx.AsyncClient` 调 TikHub 接口（绕过日志，admin_tikhub /stats 会漏统计）
- 调用方在 router/service 里把 `user_id=current_user.id` 传进去（None=系统调用如 kol_scheduler）
- 模式参考 `adapters/yunwu.py` 的 AiCallLog finally 写法（红线 #6）

### 4.3 工具类 Service

如 `intake_report.py` 的 `generate_docx()` / `generate_pdf()`，是同步文件生成函数，不涉及数据库。这类可以：
- 不接收 `db` 参数
- 返回文件路径
- 不需要 `async`

### 4.4 查询风格

```python
# 简单查询 → ORM
result = await db.execute(select(User).where(User.id == user_id))

# 复杂查询 → text()
result = await db.execute(text("""
    SELECT * FROM service_credentials
    WHERE provider = :provider AND status = 'enabled'
"""), {"provider": provider})
```

## 5. Model 模式

### 5.1 标准 Model 结构

```python
from sqlalchemy import BigInteger, Column, String, Text, TIMESTAMP, func
from sqlalchemy.dialects.postgresql import JSONB
from app.core.database import Base

class XxxModel(Base):
    __tablename__ = "xxx"

    id = Column(BigInteger, primary_key=True, autoincrement=True)
    # 业务字段 ...

    # 审计字段
    created_by = Column(BigInteger, nullable=True)
    created_at = Column(TIMESTAMP(timezone=True), nullable=False, server_default=func.now())
    updated_at = Column(TIMESTAMP(timezone=True), nullable=False, server_default=func.now())
    deleted_at = Column(TIMESTAMP(timezone=True), nullable=True)
```

### 5.2 审计字段实际情况

| 字段 | 说明 |
|------|------|
| `id` | 所有表都有，`BigInteger autoincrement` |
| `created_at` | 所有表都有，`server_default=func.now()` |
| `updated_at` | 业务表有，日志表（OperationLog/ExternalServiceLog/AiCallLog/TaskLog）没有 |
| `deleted_at` | 需要软删除的表有（User、Kol、ServiceCredential），日志表和关联表没有 |
| `created_by` | 部分表有（User、Kol、Output、TaskJob、AiCallLog），其他表没有 |

软删除查询：`.where(Xxx.deleted_at.is_(None))`

### 5.3 字段类型约定

| 类型 | 用途 | 注意 |
|------|------|------|
| `BigInteger` | 主键、外键 | 自增 |
| `String(N)` | 短文本 | 明确长度 |
| `Text` | 长文本 | 无长度限制 |
| `JSONB` | JSON 数据 | `default=list` 或 `default=dict` |
| `TIMESTAMP(timezone=True)` | 所有时间字段 | 必须带时区 |
| `Numeric` | 精确数值 | 如 credits |
| `Integer` | 普通整数 | 如 tokens |

### 5.4 updated_at 更新方式

`updated_at` 只有 `server_default=func.now()`，**没有** `onupdate`。更新时必须手动设置：

```python
values["updated_at"] = datetime.now(timezone.utc)
# 或 SQL 层面
await db.execute(update(Xxx).where(...).values(updated_at=func.now()))
```

## 8. 已知技术债

> 这些是代码现状与推荐标准不一致的地方。不要求立即重构，新代码应按推荐方式写。

| # | 问题 | 现状 | 推荐 | 涉及文件 |
|---|------|------|------|----------|
| 1 | Service 错误处理不统一 | `kol_tikhub.py` 返回 `{"status":"error"}` dict | 新 Service 用 `raise RuntimeError` | `services/kol_tikhub.py` |
| 2 | 分页参数无 Query 约束 | 直接 `page: int = 1` 无 `ge`/`le` | 新代码用 `Query(ge=1, le=100)` | 各 router |

| 4 | DB 会话模式不统一 | admin router 用 `AsyncSessionLocal()`，部分用 `Depends(get_db)` | 保持 `AsyncSessionLocal()` 一致 | 各 router |
| 5 | Service 层自行 commit | Service/Adapter 内部调用 `await db.commit()` | 待确认是否改为 Router 层统一 commit | `credential_selector.py`、`kol_tikhub.py`、`adapters/ai.py` |
| 6 | `updated_at` 无自动更新 | 没有 `onupdate`，需手动设置 | 待确认是否加 `onupdate=func.now()` | 所有 Model |
| 7 | 部分表缺审计字段 | `Credential`/`AiModel` 缺 `created_by`；日志表缺 `updated_at` | 新表按标准建齐 | 各 Model |

| 10 | `sqlalchemy.ext.asyncio` 排序 | 部分文件与 `app.*` 混排 | 归入第三方库组 | `admin_kols.py` |

## 9. 代码质量

| 规则 | 阈值 |
|------|------|
| 函数最大行数 | 50 行（超过需拆分） |
