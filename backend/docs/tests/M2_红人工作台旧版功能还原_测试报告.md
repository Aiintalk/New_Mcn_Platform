## 已验证的失败恢复

- 完整视频不支持的格式、单文件超过 500MB、本地临时文件写入异常、供应商异常、Gemini 凭证池为空：任务明确标记失败，临时文件清理，前端保留已选文件并禁用保存/导出。
- 素材视频上传失败：临时文件清理；删除与替换只处理当前红人路径下的私有对象。
- 复盘流式异常：返回错误事件，不生成伪“完成”报告。
- 千川仿写、价值观仿写、直播仿写：当前商品、红人上下文和人工复核/重试规则均有定向测试。

## 全量回归与构建已知问题

- 后端 `pytest -q` 在收集阶段被既有 `tests/intake/conftest.py` 的非顶层 `pytest_plugins` 声明阻断，未进入业务测试。八模块定向回归已通过。
- 前端 `npm run build` 仍有 20 个本分支之前的 TypeScript 类型/未使用变量错误（字幕、人格、千川合集、种草、素材库配置等）。本轮引入的价值观、千川仿写类型错误已修复。
- 前端全量测试本轮得到 295 条通过、1 条失败：`SeedingWriterPage`（种草内容仿写页）第 14 条既有测试在点击“生成种草脚本”后没有触发测试替身；相关测试与实现最后修改于本分支基线之前，且不属于本轮八模块还原改动。单独复现同样失败，已保留为基线测试债务，未越界修改无关模块。
- 本机没有 ffmpeg（视频生成工具），工作区也没有可用于真实验收的 mp4/mov 文件；未伪造视频样本。

## 外部前置条件

联合验收完整视频前，需要管理员在统一配置中完成：

1. 在管理端为固定验收库 `mcn_m1` 启用一个 provider 为 `gemini` 的模型，并在 `full_video` 配置绑定该模型；同时启用对应的 Gemini 统一凭证。
2. OSS 对象存储凭证已通过真实私有上传、读取、替换、删除验证，无需重复配置。
3. 选定同一投放素材的原片、剪辑成片各一条（mp4/mov，单条不超过 500MB）。

这些条件缺失时，系统会给出可操作错误，不会把完整视频预审悄悄替换成关键帧预审。
