## 4. request 层约定

> 文件：`src/api/request.ts`

### 4.1 基础配置

```ts
const BASE_URL = import.meta.env.VITE_API_BASE_URL ?? 'http://localhost:8000';
```

环境变量通过 `.env` 文件注入。

### 4.2 鉴权方式

每次请求通过 `buildHeaders()` 自动从 Zustand store 读取 token：

```ts
headers['Authorization'] = `Bearer ${token}`;
```

**所有 API 调用走 `get/post/patch/put/del` 封装函数，不要直接 `fetch`。**

例外：**文件下载**（需直接触发浏览器下载或 Blob）应使用原生 `fetch`，但同样需手动附加 `Authorization` header：
```ts
const res = await fetch(url, { headers: { Authorization: `Bearer ${token}` } });
```
**禁止用 `window.open(url)` 下载需鉴权的文件**（浏览器不会携带 header，会报 AUTH_TOKEN_MISSING）。

### 4.3 响应格式

后端统一返回：
```json
{ "success": true/false, "code": "...", "message": "...", "data": {} }
```

`handleResponse<T>()` 自动解包：成功时返回 `data`，失败时 `throw` 带 `code` 属性的 `Error`。

### 4.4 自动错误处理

| 错误码 | 处理行为 |
|--------|---------|
| `AUTH_TOKEN_EXPIRED` / `AUTH_TOKEN_MISSING` / HTTP 401 | `clearAuth()` + 跳转 `/login` |
| `AUTH_FORCE_CHANGE_PASSWORD` | 跳转 `/change-password` |
| `PERMISSION_DENIED` | `message.error('无权限访问')` |
| 其他业务错误 | `throw Error(msg)`，由调用方 catch 后决定是否展示 |

### 4.5 封装函数签名

```ts
get<T>(path, params?)          // params 自动过滤 undefined/''/空串
post<T>(path, body?)
patch<T>(path, body?)
put<T>(path, body?)
del<T>(path)
```

## 5. 路由与权限

> 文件：`src/App.tsx`、`src/routes/ProtectedRoute.tsx`、`src/routes/AdminRoute.tsx`

### 5.1 路由层级

```
/login                      → 公开（AuthLayout）
/intake/:token              → 公开（无 Layout）
/change-password            → 需登录（AuthLayout，无侧边栏）
/  /workspace/* /tasks /outputs → 需登录（OperatorLayout）
/admin/*                    → 需登录 + role=admin（AdminLayout）
/403  /404                  → 公开
*                           → Navigate to /404
```

### 5.2 ProtectedRoute 逻辑

1. 读取 localStorage 中的 `mcn_token`（key = `'mcn_token'`）
2. 有 token → 调用 `GET /api/auth/me` 验证
3. 验证失败 → `clearAuth()` + 跳转 `/login`
4. `mustChangePassword === true` 且不在 `/change-password` → 强制跳转 `/change-password`
5. 验证中 → 渲染「加载中...」占位

### 5.3 AdminRoute 逻辑

在 `ProtectedRoute` 之内，读取 `useAuthStore` 中的 `user.role`：
- `role !== 'admin'` → Navigate to `/403`

### 5.4 Zustand AuthStore

```ts
// 读取 token（组件内）
const token = useAuthStore(s => s.token);

// 读取 token（组件外，如 API 函数）
const token = useAuthStore.getState().token;
```

Token 持久化到 `localStorage`，key 为 `'mcn_token'`。

## 6. 布局与交互约定

> 基于 TasksPage、OutputsPage、ServiceConfigPage 等典型页面归纳。

### 6.1 页面结构模板

```tsx
<>
  {/* 1. 页面标题区 */}
  <div className="page-header">
    <div>
      <h1 className="page-title">页面标题</h1>
      <p className="page-desc">一句话描述</p>
    </div>
    <div className="page-actions">
      {/* 右侧操作按钮 */}
    </div>
  </div>

  {/* 2. 主内容（Tabs / Card） */}
  <Tabs ... />

  {/* 3. Modal（声明在最后） */}
  <Modal ... />
</>
```

### 6.2 Tabs 使用约定

- 使用 AntD `<Tabs>` 组件
- `activeKey` + `onChange` 受控
- **懒加载**：Tab 内数据在首次切换时才发请求（判断 `items.length === 0`），不在组件 mount 时全部加载

```ts
function handleTabChange(key: string) {
  setActiveTab(key);
  if (key === 'xxx' && data.length === 0) loadData();
}
```

### 6.3 长列表必须分页

- 分页使用自定义 CSS 分页组件（`.pagination` + `.page-btn`），不使用 AntD Pagination
- 默认每页 `page_size: 20`
- 当 `totalPages <= 1` 时不渲染分页条
- 状态：`page`（当前页码），`setPage` 通过上一页/下一页/页码直接跳转

### 6.4 列表状态显示约定

```tsx
{loading
  ? <div className="empty-state"><div className="empty-state-text">加载中...</div></div>
  : items.length === 0
  ? <div className="empty-state"><div className="empty-state-text">暂无数据</div></div>
  : <table className="ant-table">...</table>
}
```

### 6.5 敏感信息（API Key）显示约定

- 默认显示 `••••••••`，点击眼睛图标切换明文
- 每行维护独立 `revealed` state（组件内 `useState(false)`），不共用
- 显示明文时：`fontFamily: 'var(--font-mono)', letterSpacing: 0`
- 显示遮码时：`letterSpacing: 2`
- 使用 AntD 的 `EyeOutlined` / `EyeInvisibleOutlined` 图标

### 6.6 Modal 约定

- 详情查看：`width={600~700}`，`footer={null}`
- 报告预览：`width={800}`，`footer={null}`
- 表单提交：使用 AntD 内置 `onOk` / `okText` / `cancelText`，配合 `confirmLoading`
- `destroyOnClose` 视情况使用（需要重置内部状态时加）
- 报告/日志内容：`<pre style={{ fontFamily: 'var(--font-sans)', whiteSpace: 'pre-wrap', lineHeight: 1.8 }}>`

### 6.7 操作反馈（message）

- 成功：`message.success('...')`
- 失败：`message.error((err as Error).message || '操作失败')`
- 网络错误：`message.error('操作失败，请重试')`
- 删除使用 AntD `<Popconfirm>` 二次确认

### 6.8 时间格式

- 标准格式：`new Date(iso).toLocaleString('zh-CN')`
- 紧凑格式（列表中）：`month: '2-digit', day: '2-digit', hour: '2-digit', minute: '2-digit'`（例：`06-09 14:30`）
- 无数据时统一显示 `—`（em dash）

| `cooldown` | `badge-warning` | 冷却 |
| 其他 | `badge-gray` | 停用 |

### 6.10 聊天页面专用约定

`OperatorIntakeChatPage` 和 `IntakePage` 为全屏聊天布局，遵循：
- 外层容器：`height: 100dvh; overflow: hidden`（IntakePage）或 JS 覆盖 `.main-body`（OperatorIntakeChatPage）
- 内层：`height: 100%; display: flex; flex-direction: column`
- 消息区：`flex: 1; overflow-y: auto`
- 输入区：`flex-shrink: 0`（固定在底部）
- AI 头像：渐变 `background: linear-gradient(135deg, #7c3aed, #db2777)`（紫→粉）
- 用户气泡：`background: #7c3aed; color: #fff`
- AI 气泡：`background: var(--bg-card); border: 1px solid var(--border)`

## 7. 改动纪律

> 本节为强制要求，不得绕过。

### 动前必读

**每次开始前端改动任务前，先通读本规范文档。**

### 视觉值 Token 化

- **禁止**在 TSX / inline style / CSS 中硬编码颜色、间距、字号、圆角、阴影数值
- 所有视觉值必须引用 `variables.css` 中定义的 CSS 变量（`var(--xxx)`）
- 例外：特定组件内部的渐变色（如 AI 头像 `#7c3aed → #db2777`）属于组件私有样式，允许局部硬编码，但不得在多处重复

### 新增组件前先确认复用

新增 UI 元素前，先对照「第 3 节共享 CSS 组件清单」：
- 已有对应 CSS 类 → 直接用，不要重写
- 需要新类 → 加到 `admin.css`，并同步更新本规范的清单
- 原则：**不要同时存在 `.ant-table` 和 `.data-table` 两种表格类**（目前代码中两者共存，新页面统一用 `.ant-table`）

### 长列表必分页

凡是可能超过 15 条的列表，必须接入后端分页（`page` + `page_size`），使用 `.pagination` 组件。不得一次性加载全部数据（入驻报告列表、采集记录列表等简单列表除外，这些当前无分页，属于已知技术债）。

### 下载鉴权

需鉴权的文件下载，必须使用 `fetch` + Blob 方案，禁止使用 `window.open(url)` + query token。

### API 调用

所有 JSON API 调用必须走 `src/api/request.ts` 的封装函数（`get/post/patch/put/del`）。直接 `fetch` 仅用于文件下载。

### 改完自查清单

```
[ ] CSS 变量：没有硬编码颜色/间距
[ ] 组件复用：没有重复造轮子
[ ] 分页：列表数据有 page 参数
[ ] 错误处理：catch 块有 message.error
[ ] 鉴权下载：用 fetch+Blob，不用 window.open
[ ] 本地跑起来，目标页面正常渲染
[ ] 浏览器控制台无 TypeScript 类型报错
```
