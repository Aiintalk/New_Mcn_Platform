# M2 Sprint 24 — 前端任务：红人工作台 UI 一致性优化 v1

## 背景

用户反馈 `/kol-workspace/:kol_id` 红人工作台页面存在明显视觉问题，重点包括左侧 tab、人设仿写和种草仿写页面在工作台内的样式不协调；后续验收时进一步指出种草仿写主内容区输入框贴近卡片边框，缺少内部留白，其他写作页存在同类问题。

## 范围

- 调整 `KolWorkspacePage` 工作台外壳与左侧导航样式，使其与全站 `OperatorLayout` 的深色侧栏、橙色 active 状态保持一致。
- 调整 `PersonaWriterModule` 与 `SeedingWriterModule` 的工作台内嵌模式，解决独立页面居中布局嵌入工作台后留白过大、层级不统一的问题。
- 新增写作步骤卡片留白样式，修复 `PersonaWriterPage`、`SeedingWriterPage`、`QianchuanWriterPage`、`TiktokWriterPage` 中输入框/文本框贴近卡片边框的问题。
- 更新前端规范与 README，记录新增 CSS class 与本次布局约定。

## 不做

- 不改接口、数据结构、权限和后端逻辑。
- 不重做独立工具页整体布局；但步骤卡片内边距修正会同步作用于复用同一组件的独立写作页。
- 不引入新主题、新字体或新的品牌色。

## 验收口径

- `/kol-workspace/1` 左侧导航宽度与全站侧栏一致，长 tab 文案不拥挤。
- 人设仿写、种草仿写在工作台内左对齐展示，不再像独立页面套进工作台。
- 人设仿写、种草仿写、千川仿写、TikTok 脚本仿写的步骤卡片内部有稳定留白，输入框不再贴住卡片边框。
- 工作台相关组件测试通过，且浏览器预览无明显布局错位。
