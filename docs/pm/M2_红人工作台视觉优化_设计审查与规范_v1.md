# M2 红人工作台视觉优化：设计审查与规范 v1

## 范围与证据

范围为人物档案、产品库、千川仿写、价值观仿写、直播仿写、千川成片预审、复盘、素材库八页。修改前截图在 `frontend/artifacts/visual-polish/before/`，以 1440×900 与 1024×768 两种固定尺寸采集。

## 现状问题

1. 系统字体、圆角和表单控件尺寸在原生控件与组件库控件之间不一致。
2. 工作台背景、卡片边界与步骤条缺少稳定的中性层级，页面密集时扫描成本偏高。
3. 输入、文本框、选择器的焦点提示不统一，上传区与普通卡片的层级不够清楚。
4. 八页共享顶部、侧栏和页面标题，但留白与按钮视觉重量不够统一。

## 统一设计原则

- 采用克制的苹果官网式信息层级：浅灰工作区、白色内容面、细分割线、轻阴影。
- 橙色只用于主操作、选中和焦点，不用作大面积背景。
- 业务信息密度保持不变；不改变文案、步骤、接口、数据或流程。
- 优先处理全局变量、组件库主题、工作台外壳和共享控件，页签仅保留必要特例。

## 设计变量与公共规则

| 项目 | 规范 |
|---|---|
| 字体 | `-apple-system`、苹方与系统无衬线回退；正文 14px，说明文字 12–13px |
| 背景 | 页面 `#F5F5F7`，内容卡片白色 |
| 圆角 | 卡片与控件不超过 8px |
| 控件 | 输入与选择器 38px 高；统一浅灰边框、橙色 3px 焦点光环 |
| 卡片 | 细边框、轻阴影；卡片头 16px × 20px，正文 20px |
| 按钮 | 主按钮保留橙色；最小高度 36px；键盘聚焦可见 |
| 上传区 | 虚线边框、悬停橙色描边与浅橙背景 |

## 逐页验收标准

| 页签 | 特殊检查 |
|---|---|
| 人物档案 | 五分区标题、编辑/保存/取消的对齐与焦点 |
| 产品库 | 商品卡字段可扫描，编辑表单不拥挤 |
| 千川仿写 | 步骤条、输入区、时间线和结果区层级 |
| 价值观仿写 | 四步流程、方向卡、报告与修改历史可读 |
| 直播仿写 | 卖点步骤、锁定状态和结果操作栏清晰 |
| 成片预审 | 双视频上传卡状态、禁用与完成提示清楚 |
| 复盘 | 多文件区、解析状态、详情与导出不重叠 |
| 素材库 | 两块上传区、长正文、展开播放器与删除操作稳定 |

验收时需在两种固定尺寸检查首屏、代表性输入/焦点/结果态，确保没有横向溢出、文字截断、遮挡或不可点击控件。

## 修改后复验结论

修改后截图在 `frontend/artifacts/visual-polish/after/`，两种尺寸各八张。复验确认八页共享同一套字体、浅灰工作区、卡片边界、控件高度、焦点光环和橙色主操作层级；1024×768 下侧栏、标题、步骤条和首屏表单未出现横向溢出或按钮不可见。功能路径未改动。
