关系读法：
||--o{ 表示「1 对多」（左边 1 条，右边多条）。
核心链路： eval_versions → eval_runs → eval_case_results → eval_scores → eval_human_labels
维度侧：eval_dimensions → eval_rubrics，维度同时被 eval_scores 引用。
归一化：一个 case×run 的生成输出只存一份在 eval_case_results，多个维度的评分引用它，避免重复存储。
