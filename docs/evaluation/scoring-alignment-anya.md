## 一、这个文档是干嘛的

评价体系整体设计（策略层、评委模型、适配器、调度等）**你不用管**，一期都走默认。**你只需要定三件事**：

1. **评分维度** —— 一期千川仿写评哪几个方向（建议 3 个，见第三节）
2. **各维度权重** —— 各占多少比例（建议起步值，见第四节）
3. **评分量化表（rubric）** —— 每个维度、每个分数等级的**具体打分标准**（核心，见第五节，含模板和样例）

这三样定了，一期就能跑起来。下面的模板我都预填了**建议值/占位**，你直接改就行。

## 二、极简背景

- 评价体系 = 给千川仿写工具建一把**量化的尺子**：每改一版提示词，用固定测试集 + 你定的评分标准打分，量化看新版比旧版好了还是差了。
- 一期被测对象：**千川仿写**（脚本文案）。
- 评分方式：AI 按你定的 rubric 先打分，开发者/管理员人工抽查校准。

### v2 相比 0717 的关键变化（你需要知道的）

| 变化 | 对你的影响 |
|------|----------|
| **分场景 rubric** | 同一个维度（如文案质量），不同品类可以有**不同的评分细则**——美妆看"皮肤刺激/吸收"，减肥看"是否反弹/饮食"。你可以为每个维度定义「默认 + 若干品类变体」（见第五节）。**不强制**，一期只做默认也行。 |
| **权重可配** | 权重不写死，你随时能调（不影响结构）。 |
| 评委模型 / 策略层 / 多供应商适配器 | 一期默认，**你不用管**。 |

## 五、【安雅填写】评分量化表（rubric）—— 核心

### 5.1 量表说明

- **分数范围**：1–10 分（可在配置里改，一期建议 1–10）。
- **等级锚点**：用 5 档打分 `10 / 8 / 6 / 4 / 2`（也可改成 `10/7/5/3/1`，你定）。
- **每个等级写一句"达到这个分要看什么"**（criteria）——越具体，AI 打分越稳、越可复现。
- **分场景变体**（可选）：同一维度可按品类写多套细则（默认 + 美妆 + 减肥…），系统按业务场景挑哪套。一期只写「默认」也能跑。
